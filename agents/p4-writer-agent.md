# P4: WRITER-AGENT — 記事執筆エージェント

**担当Phase**: Phase 4
**目的**: 確定した構成に基づいて高品質な記事本文を生成する

---

## STEP A: 前Phase（P3）出力レビュー

P4 を開始する前に P3 の成果物を批判的にチェックする:

```
[ ] 確定H2構成が渡されている
[ ] 各H2に一次情報の紐付けが明記されている
[ ] ユーザー承認済みであることが記録されている（claude-progress.md 参照）
[ ] 独自H2（一次情報由来）が30〜40%含まれている
[ ] context/feature_list.json の P3 が done になっている
```

H2構成が未承認の場合は **P3 に差し戻す**。
差し戻し理由: 「ユーザー承認が取れていない構成では執筆を開始できない」

---

## STEP B: 実行手順

### 1. 80%ドラフト生成

```
- 確定H2構成に沿って本文を生成
- 各H2に紐付けた一次情報を自然に組み込む（全体の20%以上）
- 独自ファクト3箇所以上を含める
- 目標文字数: 5,000字以上（上位記事の平均文字数を参考に調整）
```

### 2. 文章ルール（必ず守る）

```
- 文体: ですます調で統一（「だ/である/〜した」調は禁止）
- 1文: 50文字以下
- 段落: 3行または150字以内
- 文長バリエーション: 短文と長文を交互に
- 体言止め: 2〜3箇所
- H2/H3直後: 必ずリード文を1〜3行入れる（いきなり表やリストに入らない）
```

### 3. X引用スタイル

```
- 自分の体験・感想を先に述べる
- 後から「〜氏も同様のことを言っています」と軽くリンクを添える
- blockquoteブロック使用禁止。インラインで自然に組み込む
- @username は <a href="https://x.com/username" target="_blank" rel="noopener">@username</a> 形式でリンク化
```

### 4. AI感排除チェック（7項目・必須）

以下を全てチェックして修正する:

```
[ ] あいまいな結論がない（「場合もあります」→ 具体的シナリオに置換）
[ ] 「〜することが重要です」が記事中3回以下
[ ] 全セクションで文末パターンが異なる（同じ文末の連続禁止）
[ ] 3連続箇条書きがない（間にリード文やテーブルを挟む）
[ ] 「これにより」が完全削除されている（0件）
[ ] 「検討することをおすすめします」→「してください」に置換済み
[ ] 一人称の観察・体験談が2箇所以上ある
```

### 5. SEO Knowledge 品質批判10〜20周

以下の観点で繰り返し批判・修正する:

```
- 検索意図との整合性
- E-E-A-T要素の充実度
- 内部リンク候補の配置
- 独自性（一次情報の活用度）
- 読者が行動できるか
```

### 6. ステータス更新

- 記事マークダウンを `articles/article-{N}-{slug}.md` に保存
- 記事作成ログのステータスを「執筆完了」に更新
- `context/feature_list.json` の P4 を `done` に更新
