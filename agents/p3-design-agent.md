# P3: DESIGN-AGENT — 記事設計エージェント

**担当Phase**: Phase 3（最重要Phase）
**目的**: 検索上位記事の構造を分析し、共通パターン＋独自パートで記事構成を確定する

---

## STEP A: 前Phase（P2）出力レビュー

P3 を開始する前に P2 の成果物を批判的にチェックする:

```
[ ] X一次情報が30件以上スプレッドシートに記録されている
[ ] X一次情報のF列に200字以上の全文が記載されている
[ ] YouTube一次情報のH列（文字起こし完了）が全件「○」
[ ] data/x_trends/ に JSONファイルが存在する
[ ] context/feature_list.json の P2 が done になっている
```

YouTube文字起こしが不足している場合は **P2 に差し戻す**。
差し戻し理由: 「YouTube文字起こし未完了。H列に×がある」

---

## STEP B: 実行手順

### 1. 検索上位10記事のH2/H3構造抽出

Ahrefs MCP を使って SERP データを取得:

```
serp-overview: 対象KW・country=JP で上位10記事のURL/DR/タイトルを取得
```

全10サイト（最低7サイト）のページをフェッチしてH2/H3見出しを抽出する。
スプレッドシートに全サイト分の見出し構造を書き出す。

### 2. 共通パターン分析

```
1. 全サイトのH2をカテゴリ分類（「とは」「分類」「選び方」「ツール一覧」「まとめ」等）
2. 各カテゴリの出現率を算出（10サイト中何サイトにあるか）
3. 共通60〜70%: 出現率50%以上のH2は踏襲（検索意図の核心）
4. 独自30〜40%: 出現率25%以下または0%のH2を一次情報から設計（差別化ポイント）
```

### 3. 一次情報の配置確定

確定H2ごとに以下を決定する:

```
- 使うX一次情報（投稿者・投稿URL・活用方法）
- 使うYouTube一次情報（動画名・タイムスタンプ・活用方法）
- メディア（画像/動画）の記事埋め込み可否
```

### 4. ユーザー確認（必須）

**確定H2構成をユーザーに提示し、承認を得てからPhase 4に進む。**

```
提示フォーマット:
  ## 記事構成（承認をお願いします）
  KW: [確定KW]
  想定文字数: [N]文字

  ### H2一覧
  1. [H2タイトル]（共通/独自）← 使う一次情報
  2. [H2タイトル]（共通/独自）← 使う一次情報
  ...

  承認いただけたら執筆を開始します。
```

自動実行モードの場合はこのステップをスキップ可。

### 5. ステータス更新

- 記事作成ログのステータスを「記事設計完了」に更新
- `context/feature_list.json` の P3 を `done` に更新

---

## STEP C: 自己評価チェックリスト

```
[ ] 検索上位10記事（最低7記事）のH2/H3構造を抽出した
[ ] 各H2の出現率（10サイト中何サイト）を算出した
[ ] 共通H2（出現率50%以上）を選定した
[ ] 独自H2（出現率25%以下）を一次情報から設計した
[ ] 共通60〜70%・独自30〜40%のバランスを確認した
[ ] 各H2に「使う一次情報」を紐付けた
[ ] ユーザーに提示して承認を得た（または自動実行モードで明記した）
[ ] 記事作成ログのステータスを「記事設計完了」に更新した
[ ] context/feature_list.json の P3 を done に更新した
```

全項目チェック後、P4: WRITER-AGENT へ確定H2構成と一次情報マッピングを渡す。
