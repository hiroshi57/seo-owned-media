# AGENTS.md — SEOオウンドメディア 7エージェント構成

このファイルはエージェントの行動規範とルーティングを定義する。
**セッション開始時に必ずこのファイルを読むこと。**

---

## 起動時の手順

1. このファイルを読む
2. `context/claude-progress.md` を読む（前回の状態を把握）
3. `context/feature_list.json` を読む（どのPhaseが未完了か確認）
4. `kpi_feedback.md` を読む（前回の成功/失敗パターン）
5. `PROJECT.md` を読む（メディア設定・ペルソナ）

---

## 評価設計の原則

各エージェントは以下の2段階評価を必ず実行してから次Phaseへ渡す。

```
STEP A: 前Phaseの出力レビュー（入力検証）
  → 前エージェントの成果物が自分の作業に足りているか批判的にチェック
  → 不足があれば前Phaseに差し戻し（差し戻し理由を明記）

STEP B: 自己評価（出力検証）
  → 自分の成果物をISCチェックリストで採点
  → 基準点に達しない場合は自己修正してから次へ渡す
```

## エージェント構成と評価基準

---

### P1: KW-AGENT（キーワード選定）

**役割**: Ahrefs APIで競合KW抽出→3C分析→SEO Knowledge批判3周→KW確定

**前Phase入力レビュー（STEP A）**: なし（パイプライン起点）

**自己評価チェックリスト（STEP B）**:
```
[ ] 確定KWのVolume ≥ 200（country: JP）
[ ] KD値が確認済みで、競合SERPにDR低サイトが存在する
[ ] parent_topicを確認してカニバリゼーションリスクがない
[ ] 3C分析（Company/Competitor/Customer）の観点で独自価値を言語化できた
[ ] SEO Knowledge批判3周を完了し、指摘への対応が記録されている
[ ] スプレッドシート「KW戦略」タブを更新した
[ ] スプレッドシート「記事作成ログ」タブに新行を追加した
```
**合格ライン**: 全項目チェック。未達項目は自己修正してから P2 へ渡す。

---

### P2: RESEARCH-AGENT（一次情報収集）

**役割**: X API v2で関連投稿30件+YouTube文字起こし5-10本の収集

**前Phase入力レビュー（STEP A）**:
```
[ ] 確定KWがP1から渡されている
[ ] KW戦略タブにKWが記録されている
[ ] 記事作成ログに該当行が存在する
```
→ 未達なら P1 に差し戻し。差し戻し理由: 「確定KWが未記録のため一次情報収集を開始できない」

**自己評価チェックリスト（STEP B）**:
```
[ ] X API v2で30件以上の投稿を収集した（min_impressions: 500）
[ ] note_tweet（全文）、投稿URL、メディアURLを取得した
[ ] YouTube動画を5本以上収集し、yt-dlpで字幕取得した
[ ] data/x_trends/x_enriched_article{N}.json に保存した
[ ] スプレッドシート「X一次情報」タブを記録した（F列200字以上の全文）
[ ] スプレッドシート「YouTube一次情報」タブを記録した
[ ] YouTube一次情報タブのH列（文字起こし完了）が全件「○」
```
**合格ライン**: 全項目チェック。API 503が3回連続の場合のみエラーログに記録して P3 へ進む（その旨を明記）。

---

### P3: DESIGN-AGENT（記事設計）

**役割**: 検索上位10記事のH2構造分析→共通60-70%+独自30-40%の構成確定

**前Phase入力レビュー（STEP A）**:
```
[ ] X一次情報が30件以上スプレッドシートに記録されている
[ ] YouTube一次情報のH列（文字起こし完了）が全件「○」
[ ] 一次情報の質（F列に200字以上の本文がある）を確認
```
→ YouTube文字起こしが不足なら P2 に差し戻し。差し戻し理由を明記。

**自己評価チェックリスト（STEP B）**:
```
[ ] 検索上位10記事（最低7記事）のH2/H3構造を抽出した
[ ] 各H2の出現率（10サイト中何サイト）を算出した
[ ] 共通H2（出現率50%以上）を選定した
[ ] 独自H2（出現率25%以下 or 0%）を一次情報から設計した
[ ] 共通60-70%、独自30-40%のバランスを確認した
[ ] 各H2に「使う一次情報」を紐付けた
[ ] H2構成をユーザーに提示して承認を得た（自動実行モード除く）
[ ] 記事作成ログのステータスを「記事設計完了」に更新した
```
**合格ライン**: ユーザー承認が必須（自動実行モード以外）。全項目チェックで P4 へ。

---

### P4: WRITER-AGENT（記事執筆）

**役割**: 80%ドラフト→一次情報20%組み込み→AI感排除→品質批判10-20周

**前Phase入力レビュー（STEP A）**:
```
[ ] 確定H2構成が渡されている
[ ] 各H2に一次情報の紐付けが明記されている
[ ] ユーザー承認済みであることが確認できる（claude-progress.md に記録あり）
[ ] 設計に独自H2（一次情報由来）が30-40%含まれている
```
→ H2構成が未承認なら P3 に差し戻し。差し戻し理由: 「ユーザー承認が取れていない」

**自己評価チェックリスト（STEP B）**:
```
[ ] 本文が5,000文字以上ある
[ ] 一次情報（X/YouTube）が全体の20%以上に組み込まれている
[ ] 独自ファクトが3箇所以上ある
[ ] AI感排除7項目を全てチェックして修正した（下記参照）
[ ] ですます調で統一されている
[ ] 1文50文字以下・段落3行以内
[ ] H2/H3直後に必ずリード文がある
[ ] 「これにより」が0件
[ ] 「検討することをおすすめします」が0件
[ ] 一人称の体験談・観察が2箇所以上ある
[ ] 品質批判を10周以上実施した
[ ] 記事作成ログのステータスを「執筆完了」に更新した
```

**AI感排除7項目**:
```
1. あいまいな結論がない（「場合もあります」を具体シナリオに置換）
2. 「〜することが重要です」が3回以下
3. 全セクションで文末パターンが異なる
4. 3連続箇条書きがない
5. 「これにより」が完全削除されている
6. 「検討することをおすすめします」が削除されている
7. 一人称の観察・体験談が2箇所以上ある
```
**合格ライン**: AI感排除7項目全通過。品質批判10周完了。

---

### P5: QA-AGENT（品質チェック）

**役割**: 5視点エージェント採点（各20点満点）、95点以上で公開承認

**前Phase入力レビュー（STEP A）**:
```
[ ] 記事本文が5,000文字以上ある
[ ] AI感排除7項目チェック済みの記録がある
[ ] ですます調統一・段落ルール準拠を確認
[ ] 一次情報が20%以上組み込まれている
```
→ 文字数不足・AI感未排除なら P4 に差し戻し。差し戻し理由を明記。

**自己評価チェックリスト（STEP B）**:
```
[ ] デザインエージェント採点（/20点）を完了
[ ] SEOエージェント採点（/20点）を完了
[ ] 編集者エージェント採点（/20点）を完了
[ ] 技術専門家エージェント採点（/20点）を完了
[ ] ペルソナエージェント採点（/20点）を完了
[ ] 合計スコアが95点以上
[ ] 95点未満の場合: 修正ループを最大3サイクル実施した
[ ] 3サイクル後も95点未満の場合: ユーザーに差分を報告した
[ ] 記事作成ログに品質スコアを記録した
[ ] 記事作成ログのステータスを「品質チェック完了」に更新した
```
**合格ライン**: 合計95点以上。

採点基準の詳細は `SEOオウンドメディア自動化_セットアップ.md` セクション4 Phase 5を参照。

### P6: PUBLISH-AGENT（公開）

**役割**: 画像生成→WordPress投稿→Rank Math設定→公開

**前Phase入力レビュー（STEP A）**:
```
[ ] 品質スコアが95点以上であることを確認
[ ] context/feature_list.json のP5 statusが「done」になっている
[ ] quality_scoreフィールドに数値が記録されている
```
→ 品質スコア未達なら P5 に差し戻し。差し戻し理由: 「品質スコアが95点未満」

**自己評価チェックリスト（STEP B）**:
```
[ ] アイキャッチ画像を生成した（16:9、英語テキストなし確認済み）
[ ] H2直下図解を3-5枚生成した（英語テキストなし目視確認済み）
[ ] 全画像のaltテキストを設定した
[ ] Markdown→HTML変換が正常に完了した
[ ] WordPress に draft で投稿した（wp_post_id を取得）
[ ] 画像をWordPressにアップロードしてfeatured_mediaに設定した
[ ] Rank Math でSEOタイトル・メタディスクリプション・フォーカスKWを設定した
[ ] FAQ Schemaを5問以上設定した
[ ] ユーザー確認後に status: publish に変更した（自動実行モードは95点以上で自動公開）
[ ] 記事作成ログのステータスを「公開済み」に更新した
[ ] WP Post IDを記録した
```
**合格ライン**: 全項目チェック。画像の英語混入があれば再生成してから進む。

### P7: ANALYTICS-AGENT（分析）

**役割**: インデックス登録→スプレッドシート全タブ更新→内部リンク追加→KPIレポート→kpi_feedback.md更新

**前Phase入力レビュー（STEP A）**:
```
[ ] WordPress公開URL が存在する
[ ] 記事作成ログのステータスが「公開済み」になっている
[ ] WP Post ID が記録されている
[ ] 品質スコアが記録されている
```
→ 公開URLがなければ P6 に差し戻し。差し戻し理由: 「公開URLが取得できていない」

**自己評価チェックリスト（STEP B）**:
```
[ ] Google Indexing APIにインデックス登録リクエストを送信した
[ ] スプレッドシート「ダッシュボード」タブを更新した
[ ] スプレッドシート「記事作成ログ」タブのURLと品質スコアを記録した
[ ] スプレッドシート「KW戦略」タブのStatusを「公開済」に更新した
[ ] スプレッドシート「トピッククラスター」タブのStatusとURLを更新した
[ ] スプレッドシート「KPIレポート」タブに新行を追加した
[ ] 既存記事への内部リンクを自動追加した（3本以上）
[ ] 「内部リンク管理」タブに記録した
[ ] kpi_feedback.md の成功パターン/失敗パターンを更新した
[ ] context/MEMORY.md に今回の学びを追記した（2,000字上限）
[ ] context/claude-progress.md にセッション記録を追記した
[ ] context/feature_list.json の全Phaseをリセット（次記事に備えて pending に戻す）
```
**合格ライン**: kpi_feedback.md と MEMORY.md の更新まで完了して1サイクル完了。

## Orchestratorのルーティング優先度

```
優先度 | 条件                                           | 次のエージェント
1      | context/feature_list.json が存在しない          | INIT（セットアップを促す）
2      | .env に未設定APIキーがある                      | PAUSE_FOR_HUMAN（設定依頼）
3      | 記事作成ログの「未着手」行がある                   | P1: KW-AGENT
4      | P1完了済み・P2未完了                            | P2: RESEARCH-AGENT（+ P1の出力レビュー）
5      | P2完了済み・P3未完了                            | P3: DESIGN-AGENT（+ P2の出力レビュー）
6      | P3完了済み・P4未完了                            | P4: WRITER-AGENT（+ P3の出力レビュー）
7      | P4完了済み・P5未完了                            | P5: QA-AGENT（+ P4の出力レビュー）
8      | P5完了済み・P6未完了                            | P6: PUBLISH-AGENT（+ P5の出力レビュー）
9      | P6完了済み・P7未完了                            | P7: ANALYTICS-AGENT（+ P6の出力レビュー）
10     | 全Phase完了済み                                | 次記事のKW選定へ or スケジュール待機
```

---

## 越境禁止ルール

- **1フェーズ = 1エージェントの責任範囲**。他フェーズの判断を勝手に行わない
- Phase 3完了後は必ずユーザーにH2構成を提示して承認を得てからPhase 4へ進む
- 承認なしにWordPress公開（status: publish）は実行しない
- 差し戻しは1段階のみ（P4→P3 はOK。P4→P2 は不可。P3に差し戻してP3がP2を再度評価する）

## 自己学習ループ（AIを自動で育てる）

P7: ANALYTICS-AGENTが毎回実行するAI育成サイクル:

```
1. 今回の記事の品質スコア・順位・表示回数を確認
2. 品質スコア97点以上の記事 → その構造的特徴をMEMORY.mdに追記
3. 7日以内にGSC表示が発生した記事 → KW特性をMEMORY.mdに追記
4. 95点に届かなかった指摘事項 → 失敗パターンとしてkpi_feedback.mdに追記
5. 次回のKW-AGENTがMEMORY.mdとkpi_feedback.mdを参照して選定基準を自動調整
```
