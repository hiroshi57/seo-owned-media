# スプレッドシート構成（詳細）

10タブの列定義と運用ルールを記載する。

---

## KW戦略タブ
3C分析の結果を記録し、KWの全体戦略を管理する。
```
A: KW（キーワード）
B: Volume（月間検索ボリューム）
C: KD（キーワード難易度）
D: Intent（検索意図: info / commercial / transactional）
E: CV距離（Do > Buy > Commercial > Info の順でスコアリング）
F: 自社独自価値（このKWで書ける独自情報の概要）
G: Status（未着手 / 執筆中 / 公開済 / リライト中）
H: 備考
```

---

## トピッククラスタータブ
クラスター構造を管理し、ピラー記事とクラスター記事の関係を可視化する。
```
A: KW
B: Volume
C: KD
D: カテゴリ（サイトのカテゴリ分類）
E: クラスター名（所属するトピッククラスター）
F: 種別（ピラー / クラスター）
G: 優先度（A / B / C）
H: Status（未着手 / 公開済）
I: URL（公開後に記入）
```

---

## X一次情報タブ
X API v2で収集した投稿データを記事ごとに管理する。
```
A: 記事#（記事番号）
B: KW
C: 投稿者名（@username）
D: 投稿日
E: 投稿URL
F: 本文（note_tweet全文。200字以上）
G: インプレッション数
H: RT数
I: Like数
J: メディアURL
K: 記事への活用方法（どのH2でどう使うか）
```
**注意**: 記事間に空行を入れない。#1→#2→#3と隙間なく詰めて記録する。

---

## YouTube一次情報タブ
YouTube Data APIで収集した動画データと文字起こしを管理する。
```
A: 記事#
B: KW
C: 動画タイトル
D: チャンネル名
E: 動画URL
F: 再生数
G: 文字起こしファイルパス
H: 文字起こし完了（○ / ×）
I: 要約（動画内容の要点）
J: 記事への活用方法
```
**注意**: H列が全件「○」になるまでPhase 4に進まない。

---

## ダッシュボードタブ
サイト全体のサマリーを1画面で確認できるようにする。
```
A: 指標名
B: 値
C: 前日比
D: 備考
```
主な指標: 累計公開記事数 / ドメインレーティング / 被リンク数 / オーガニックKW数 / 推定トラフィック / インデックス済み記事数 / 今月公開数

---

## 記事作成ログタブ
全記事の制作履歴と品質管理を行う。
```
A: 記事#
B: 制作日
C: KW
D: タイトル
E: URL
F: カテゴリ
G: Volume
H: 品質ループ回数
I: 品質スコア（5エージェント合計）
J: ステータス（未着手 / KW選定中 / 一次情報収集中 / 執筆中 / draft / レビュー中 / 公開済み）
K: WP Post ID
L: 備考
```
**注意**: パイプラインは必ずこのタブの「未着手」行の一番上から順番に開始する。

---

## KPIレポートタブ
日次のKPIデータを蓄積する。
```
A: 日付
B: 総公開記事数
C: 本日公開数
D: インデックス率（%）
E: PV（GA4）
F: 表示回数（GSC）
G: クリック数（GSC）
H: CTR（GSC）
I: 平均順位（GSC）
J: 備考
```

---

## 内部リンク管理タブ
全内部リンクを記録し、リンク漏れを防止する。
```
A: リンク元記事（タイトル or URL）
B: リンク先記事（タイトル or URL）
C: アンカーテキスト
D: 設置日
E: リンク先URL
```

---

## リライトログタブ
リライト履歴を記録し、効果測定に活用する。
```
A: 記事#
B: リライト日
C: リライト理由（順位改善 / 情報更新 / カニバリ対応 / 内部リンク追加）
D: 変更箇所の概要
E: リライト前順位
F: リライト後順位（1週間後に記入）
G: 効果（改善 / 変化なし / 悪化）
```

---

## エラーログタブ
パイプライン実行中のエラーを記録する。
```
A: 日付
B: Phase（1-7）
C: エラー内容
D: 対応内容
E: 解決状態（解決済み / 未解決 / 回避策適用）
```
